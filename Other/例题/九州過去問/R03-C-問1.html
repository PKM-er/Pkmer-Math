3. 非交换语言的分析

结合 L(M)L(M)L(M) 与交换语言的定义：

  • 若 LLL 为含 aaa 偶数个的语言，则其交换语言含 bbb 偶数个。
  • 例如，图1.2 和图1.4 中的语言 L(M)L(M)L(M) 是交换的；而图1.3 中的语言不满足交换条件，因为 bbb 的出现次数受限。

where

Diagram (the second one in Figure 1)

  • States: q0q_0q0​ (final), q1q_1q1​.

Let’s see what strings cause the DFA to end in q0q_0q0​.

  1. Reading an aaa in q0q_0q0​ keeps us in q0q_0q0​.
  2. Reading a bbb in q0q_0q0​ moves us to q1q_1q1​.
  3. Reading an aaa in q1q_1q1​ stays in q1q_1q1​.
  4. Reading a bbb in q1q_1q1​ moves us back to q0q_0q0​.

Equivalently, the DFA flips between q0q_0q0​ and q1q_1q1​ only on input bbb. Every time we see a bbb, we switch states; every time we see an aaa, we remain in the same state.

  • We start in q0q_0q0​.
  • If we read an even number of bbbs overall, we end up back in q0q_0q0​.
  • If we read an odd number of bbbs overall, we end in q1q_1q1​.

Thus the machine accepts exactly those strings having an even number of bbbs (with no restriction on how many aaas appear, or where). Formally:

Diagram (the third one in Figure 1)

Now let’s track the effect of reading the last character:

  • From q0q_0q0​, reading bbb brings us to q1q_1q1​.
  • From q1q_1q1​, reading bbb keeps us in q1q_1q1​.
  • From q1q_1q1​, reading aaa brings us back to q0q_0q0​.
  • From q0q_0q0​, reading aaa keeps us in q0q_0q0​.

One very direct way to see what is accepted:

  • If a string ends with aaa, then the last transition will be “q1→aq0q_1 \xrightarrow{a} q_0q1​a​q0​” (if we were in q1q_1q1​ before) or “q0→aq0q_0 \xrightarrow{a} q_0q0​a​q0​” (if we were already in q0q_0q0​ before). In either case, that final aaa puts (or keeps) us in q0q_0q0​.
  • If a string ends with bbb, we must be in q1q_1q1​ at the end (since from q0q_0q0​ reading b→q1b\to q_1b→q1​, or from q1q_1q1​ reading b→q1b\to q_1b→q1​). That is not accepted, because q1q_1q1​ is not final.

So the machine exactly accepts all strings that are either empty or end in aaa. Formally:

Diagram (the fourth one in Figure 1)

Notice what happens from q1q_1q1​: any input (whether aaa or bbb) returns you to q0q_0q0​. Equivalently:

  • From q0q_0q0​, reading aaa stays in q0q_0q0​.
  • From q0q_0q0​, reading bbb jumps to q1q_1q1​.
  • Then from q1q_1q1​, the very next character (be it aaa or bbb) sends us back to q0q_0q0​.

We cannot remain in q1q_1q1​ for more than one symbol. In particular, we cannot end in q1q_1q1​ unless the input string ends immediately after a bbb read from q0q_0q0​. But ending in q1q_1q1​ is a rejection. So any time we read a bbb from q0q_0q0​, we must read at least one more character to get back to q0q_0q0​.

A convenient description of L(M4)L(M_4)L(M4​) is via a regular expression:

L(M4)  =  (a∗) (b(a∣b) a∗)∗ ⁣.L(M_4) \;=\; \bigl(a^*\bigr)\,\bigl(b(a\mid b)\,a^*\bigr)^*\!. L(M4​)=(a∗)(b(a∣b)a∗)∗.

In words:

  1. We can read any number of aaas (staying in q0q_0q0​).
  2. If we read a bbb from q0q_0q0​, we jump to q1q_1q1​; then the next symbol (which must exist) returns us to q0q_0q0​. After that, again any number of aaas is allowed, etc.

Equivalently, one cannot end the string in “q1q_1q1​”, so one cannot have the last symbol be a “bbb” from q0q_0q0​. But one can have arbitrarily many pairs “b,xb, xb,x” in the middle (where xxx is aaa or bbb), each pair returning us to q0q_0q0​.

  • Start in q0q_0q0​.

  • Otherwise first symbol →q1\to q_1→q1​.

3. “Flipped language” discussion

Finally, part (3) involves the notion of flipped language:

For L=∥e(a,b)∥L=\|e(a,b)\|L=∥e(a,b)∥, we call ∥e(b,a)∥\|e(b,a)\|∥e(b,a)∥ the flipped language of LLL.

They give an example: “strings with an even number of aaas” flips to “strings with an even number of bbbs.”

The question says:

For each MMM with δ\deltaδ different from those in (1) or (2), describe the strings in L(M)L(M)L(M) which is not the flipped language of a language in (1) or (2).

Put differently, once you exhaust the 2-state DFAs from (1) and (2), you’ll find there are still a few other 2-state DFAs possible (because you can also choose which state is final in different ways). If one of those left-over DFAs is actually the “flip” of one in (1) or (2), we don’t need to re-describe it. But if it is not just a flip of one we already covered, then we do describe it.

One typical example:

  • “Strings that contain no aaa” flips to “Strings that contain no bbb.”
  • “Strings with an even number of bbb” flips to “Strings with an even number of aaa.”
  • “Strings ending in aaa” flips to “Strings ending in bbb.”
  • etc.

Thus the question is prompting you to check all possible 2-state automata, identify which ones are “the same or the flip” of the ones already enumerated, and for any that are not flips, give the language description.

In many textbook exercises, once you list out all 2-state DFAs (with one final state, or with both states final, etc.), you notice each one’s language is either

  • “all strings” or “no strings,”
  • “only a∗a^*a∗,” or “only b∗b^*b∗,”
  • “even number of bbbs,” “even number of aaas,”
  • “ends in aaa,” or “ends in bbb,”
  • “the empty string {ϵ}\{\epsilon\}{ϵ},” or “all nonempty strings,” etc.

Many of those can be matched up as flips of each other. Whichever remain unmatched are precisely the ones the question is targeting.

Concluding Remarks

The key takeaway is that all 2-state DFAs over {a,b}\{a,b\}{a,b} either accept a very simple “finite index” property (like “even number of bbbs”) or “no bbb” or “ends in aaa” etc. Part (3) is basically a bookkeeping exercise to show that if you try any other configuration of transitions/final-states, you get one of those old languages or you get a “flipped” version (swapping aaa and bbb throughout) of one of those languages. The ones that are truly “new” you must list separately.
