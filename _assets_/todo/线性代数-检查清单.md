# 线性代数 内容规范检查清单

目的：按 `Other/说明/内容规范.md` 的规则检查并修正 `线性代数/` 下的文档。

检查项（逐文件）
1. 文件类型识别
   - 目录文件：文件名包含 `-` 前后（例如 `--线性代数--.md`）
   - 词条（Entry）：常位于三级目录，与目录文件同级；不应包含任何 Markdown 标题（`#`）
   - 文档（Document）：可以包含标题结构（`##`, `###` 等）
2. YAML frontmatter
   - 目录文件应包含 `tags: [数学目录]` 和 `dlink` 字段
   - 文档建议包含 `tags`, `dlink`, `aliases`（如适用）
3. 词条格式
   - 不允许任何等级标题（含 `#`）
   - 核心内容应放在一个 Callout（`> [!NOTE] 标题`）或直接以公式/定义段落呈现
4. 目录文件内容
   - 包含章节导语（≤200字）
   - 包含 `dataview` 动态目录示例
5. 模板一致性
   - 新文件应参照 `Template/` 中的模板格式（如 frontmatter 格式）

修正策略
- 严重偏离（缺失目录 frontmatter、词条包含标题等）：自动修正并备份原文（在同目录创建 `.bak`）。
- 模糊/需人工判断项：记录在 `线性代数-检查记录.md` 中并跳过自动修改。
